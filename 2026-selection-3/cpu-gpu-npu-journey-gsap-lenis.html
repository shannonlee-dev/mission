AI 업무는 왜 CPU에서 GPU를 거쳐 NPU로 갔을까

칩의 진화가 아니라, 무대의 재설계

AI 워크로드의 병목은 단순한 연산 횟수보다 제어 비용, 병렬 처리 규모, 데이터 이동 거리에서 갈린다. CPU, GPU, NPU의 차이는 같은 수식을 어떤 실행 구조와 메모리 구조로 처리하느냐에 있다.

CUT 5
GPU는 행렬곱을 블록별로 나누어 수행한다
이 장면은 GPU가 전체 행렬곱을 타일 단위로 분할한 뒤, 각 타일에서 동일한 곱셈-누산 패턴을 병렬 반복해 전체 출력을 조합하는 과정을 8x8과 4x4 예시로 축약해 보여준다.
CUT 6
GPU는 자주 되돌아가고, NPU는 안에서 이어간다
이 장면은 GPU에도 레지스터와 캐시 같은 메모리 계층이 있지만 연산 블럭 사이에서 데이터를 다시 불러오는 왕복이 더 잦고, NPU는 온칩 저장공간과 PE 가까운 경로 안에서 데이터를 더 오래 이어 쓰도록 설계된다는 차이를 단순화해 보여준다.
CUT 7
컴파일러가 흐름을 미리 짠다
이 도식은 이미 존재하는 PE, NPU 저장공간, 일반 저장공간, 출력 C 위에 NPU 컴파일러가 A와 W의 배치, PE 실행 순서, 출력 저장 목적지를 순서대로 덧입혀 가는 모습을 핵심만 추려 보여준다.
CUT 8
NPU는 파동처럼 계산한다
이 장면은 시스톨릭 어레이에서 입력 활성값 A와 가중치 W가 서로 다른 축으로 주입되어 각 PE에서 곱셈-누산을 수행하고, 누적 결과가 대응하는 출력 C 원소로 정착하는 데이터플로우를 3x3 예시로 축약한 것이다.
