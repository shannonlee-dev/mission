한눈에 보는 구조

이 프로그램은 “사람이 입력한 Redis식 명령을 파싱해서, 직접 만든 자료구조에 값을 넣고 빼는 작은 메모리 저장소”입니다. 아래 박스를 누르면 발표용 설명이 바뀝니다.

Entry

mini_redis.py

실행 파일입니다. 사용자가 `python3 mini_redis.py`를 실행하면 `MiniRedisCLI`를 만들고 REPL을 시작합니다.

한 문장 요약

Mini Redis는 컴퓨터 안에 잠깐 보관하는 메모장이고, 키 이름으로 값을 바로 찾고, 오래 안 쓴 값이나 시간이 지난 값을 자동으로 치웁니다.

핵심 성취

빠른 저장소의 핵심 부품인 해시맵, 이중 연결 리스트, 최소 힙을 내장 `dict`, `set`, `collections` 없이 직접 구현했습니다.

범위 밖으로 둔 것

네트워크 서버, 파일 저장, Redis의 복잡 자료형, 동시성 처리는 구현하지 않았습니다. 이 과제는 내부 동작 학습에 집중합니다.

명령이 지나가는 길

명령 버튼을 누르면 사용자가 입력한 한 줄이 어떤 단계로 처리되는지 바뀝니다. 발표 중에는 `SET`, `GET`, `EXPIRE`, `CONFIG` 네 개만 보여줘도 핵심이 충분히 전달됩니다.

내부 자료구조

Store는 혼자 일하지 않습니다. 찾기, 최근 사용 순서, 만료 시간 관리를 서로 다른 자료구조에 맡깁니다.

HashMap: 빠른 찾기

키를 해시 함수로 숫자로 바꾸고, 해당 버킷에서 값을 찾습니다. 충돌은 이중 연결 리스트로 이어 붙입니다.

0 1 2 user:2 3

파일: `hash_map.py`, `linked_list.py`

LRU List: 최근 사용 순서

앞쪽은 최근 사용, 뒤쪽은 오래된 키입니다. 메모리가 넘치면 뒤에서부터 제거합니다.

최신 user:3 -> user:2 -> 오래됨 user:1

파일: `linked_list.py`, `store.py`

MinHeap: 만료 알람

가장 빨리 만료될 키가 맨 위에 옵니다. 명령 실행 전에 만료 시간이 지난 키를 먼저 정리합니다.

1003 temp 위 1010 session 1020 cache

파일: `min_heap.py`, `store.py`

DynamicArray: 보너스 배열

공간이 가득 차면 용량을 2배로 늘리는 동적 배열입니다. 자료구조 확장 개념을 보여주는 보너스 구현입니다.

0 1 2 3 가득 차면 8칸

파일: `dynamic_array.py`

Tree / BST: 보너스 트리

전위, 중위, 후위, 레벨 순회와 BST 삽입/탐색/삭제를 구현했습니다. 힙의 트리 관점을 보강합니다.

4 / \ 2 6

파일: `tree.py`

Pub/Sub: 보너스 메시징

채널별 구독자 수와 메시지 목록을 관리합니다. 메시지 흐름을 큐처럼 설명할 수 있습니다.

news -> hello update

파일: `store.py`, `cli.py`

60초 설명 대본

  1. 사용자는 `mini-redis>` 프롬프트에 `SET`, `GET`, `EXPIRE` 같은 명령을 입력합니다.
  2. `cli.py`가 이 한 줄을 단어로 나누고, Redis 스타일 규칙에 맞는지 확인합니다.
